# Architecture Decisions

| Decision        | Choice                           | Rationale                                                                                                                                                                                                                  |
| --------------- | -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| AI Model        | Gemini 2.5 Flash over 2.0 Flash  | 2.0 Flash is being deprecated June 2026. 2.5 Flash has better vision, structured output, and the same free tier.                                                                                                           |
| Cache           | Upstash Redis over Cloudflare KV | Redis gives us sorted sets (leaderboards), pub/sub, atomic counters, and more flexible data structures than KV's simple key-value. KV is still used for static assets via CF Workers.                                      |
| ML Models       | No custom ML models              | Gemini Vision + Google Cloud Vision covers all image recognition needs. Custom ML adds complexity with no benefit at this scale.                                                                                           |
| Background Jobs | Upstash QStash                   | CF Workers can't run long processes. QStash HTTP webhook-based queuing handles async tasks (Reddit fetch, TripAdvisor intel prefetch, image verification).                                                                 |
| Architecture    | Single repo (drift-api)          | All backend logic in one Hono app deployed as a single CF Worker. Route-based code splitting keeps it organized. No microservices overhead.                                                                                |
| Auth            | Better Auth                      | Framework-agnostic TypeScript auth library. Handles email/password, Google OAuth, sessions, password reset, account deletion out of the box. Auto-manages its own DB tables. Plugin ecosystem for 2FA, rate limiting, etc. |
| Vector Search   | pgvector for semantic search     | PostgreSQL pgvector extension. Embeddings stored alongside trip data. No external vector DB needed.                                                                                                                        |
| File Storage    | S3 MinIO for file storage        | Photos, PDFs, exports. Self-hosted, S3-compatible API. Full control over storage.                                                                                                                                          |
| Personalization | Dynamic AI-first, no templates   | Every itinerary is generated fresh with user-specific context (travel DNA, past trips, feedback). Destination research is shared raw material, but the output is always personalized.                                      |
| Trip Memories   | AI-generated post-trip summaries | Completed trips get an AI narrative summary, highlight picks, and stats. Feeds back into Travel DNA for future personalization. Turns the app into a travel journal.                                                       |
